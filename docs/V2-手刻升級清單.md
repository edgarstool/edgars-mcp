# mcp-handcraft → MCP v2（2026-07-28）手刻升級清單

原則：**維持手刻**，不改用官方 SDK 套件；用 48 篇官方文檔當「規範檢查表」，
把你自己的 `server_http.py` 一項項升到 2026-07-28 世代，且**持續相容舊客戶端**。

## ✅ 這批已做（feat/v2-2026-upgrade 分支）
1. **授權全綠**：`_ensure_mcp_request_authorized` 改責任鏈——CF Access 或 Bearer
   任一有效即放行（原本主網域上兩者互斥、誤擋正確 Bearer）。
2. **開啟 2026-07-28 世代**：新增 `server/discover` 方法。這是關鍵——原本缺這個，
   害現代客戶端探測失敗、被迫退回舊世代 2025-11-25。現在現代客戶端會落在 2026-07-28。
3. **版本協商**：`SUPPORTED_PROTOCOL_VERSIONS = [2026-07-28, 2025-11-25]`；
   `initialize` 仍服務舊客戶端（雙世代並存，零設定）。
4. **快取提示**：`tools/list` 與 `server/discover` 回傳 `ttlMs` / `cacheScope`，
   讓客戶端/閘道少打幾次、更快。
5. **initialize 補上 instructions**、capabilities 統一用 `build_server_capabilities()`。

## 🔜 建議下一批（可選，依你要不要那些能力）
每項都標了對應的官方文檔章節，維持手刻即可照做：

| 項目 | 為什麼 / 何時要 | 對應文檔 |
|---|---|---|
| `_meta` 信封處理 | 2026 每個請求帶 `_meta`（progress token、trace）；至少要能安全略過（已 OK），要用進度就得讀它 | handlers/progress、migration |
| 結構化輸出全面化 | 你已有 `structuredContent`/`outputSchema` helper，可讓更多工具回結構化資料 | servers/structured-output |
| 錯誤碼規範 | 未知方法回 `-32601`、參數錯 `-32602`、內部錯 `-32603`；工具「可讓模型重試的錯」用 `isError:true` 而非協定錯 | servers/handling-errors |
| `completion/complete` | 給 UI 自動補參數值（如路徑、repo 名） | servers/completions |
| `subscriptions/listen` | 想做「資源變更即時通知」時 | handlers/subscriptions |
| resources / prompts 基本類 | 想暴露「可讀資料」與「提示範本」給客戶端時 | servers/resources、prompts |
| 多 worker 無狀態確認 | 你目前是無狀態請求/回應，天生好水平擴充；上線多 worker 時確認不要引入 session | run/deploy、run/legacy-clients |

- SSE 舊傳輸、client→server progress → 新建不碰。
