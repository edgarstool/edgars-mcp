# MCP 自動巡邏（新手版）

德德，這份文件告訴你：**電腦會自己幫你顧 MCP**，你不用每次出問題才叫 AI 來修。

## 一句話

裝好排程之後，Windows 會像鬧鐘一樣定時：

1. **檢查** MCP 有沒有活著  
2. **壞了才修**（不是一直亂重啟）  
3. **你開機登入**時自動把服務拉起來  

## 你只要做一次的事

請 AI 或自己跑這行（通常要按「是」給管理員權限）：

```powershell
cd V:\projects\mcp-handcraft
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-patrol-scheduler.ps1
```

裝好就不用管 git、不用懂 commit。**裝一次就好**，之後可以重複跑（會自動更新排程設定）。

## 電腦會自動做什麼？

| 名字 | 多久跑一次 | 做什麼 | 你會感覺到嗎 |
|------|-----------|--------|-------------|
| **MCP-HealthCheck** | 每 5 分鐘 | 檢查 MCP 五層健康（程序、port、網路、tunnel） | 不會，背景靜悄悄 |
| **MCP-AutoRecover** | 每 15 分鐘 | **連續壞 3 次**才重啟 MCP；Linear webhook 服務（:8645）掛了會幫開 | 偶爾 MCP 閃一下重連 |
| **MCP-Maintain-Daily** | 每天凌晨 4 點 | 完整維護、不健康就重啟 | 睡覺時處理，醒來通常已好 |
| **MCP-LoginStart** | 你登入 Windows 時 | 啟動 MCP + linear-orchestrator | 登入後幾十秒內服務就緒 |

## 怎麼知道它有在運作？

### 方法一：看日誌（最簡單）

打開資料夾：

`V:\projects\mcp-handcraft\logs\`

找今天日期的檔案，例如：

- `patrol-healthcheck-20260704.log` — 每 5 分鐘會多幾行  
- `patrol-recover-20260704.log` — 有修復時會寫  
- `patrol-login-20260704.log` — 登入時會寫  

**有在更新時間戳 = 排程有在跑。**

### 方法二：看健康摘要

`G:\AI_WORK_512\run\mcp-handcraft\healthcheck-summary.json`

- `overallOk: true` → 目前 OK  
- `consecutiveFails` → 連續失敗幾次（到 3 會觸發自動重啟）

### 方法三：桌面捷徑（你想親眼確認時）

雙擊桌面 **「檢測MCP」** 捷徑，會跑完整檢查報告。  
**平常不用每天點** — 排程已經在顧了。

### 方法四：工作排程器（進階）

1. 按 Win 鍵，搜尋「工作排程器」  
2. 左邊點「工作排程器程式庫」  
3. 找資料夾 **Edgars**  
4. 四個任務狀態應為 **就緒** 或 **執行中**

## 什麼時候才需要叫 AI？

| 情況 | 要不要叫 AI |
|------|------------|
| MCP 偶爾斷一下又自己好 | **不用** — AutoRecover 在處理 |
| 日誌一直在寫、summary 大多 OK | **不用** — 正常 |
| 連續好幾天 `overallOk: false` | **要** — 可能有 DNS、tunnel、Doppler 等設定問題 |
| 重開機後完全沒 log 更新 | **要** — 排程可能沒裝成功或沒登入 |
| 想加新功能、改架構 | **要** — 這超出自動巡邏範圍 |

## 跟 Cursor / Copilot「自動化」的差別

- **這套排程（主要）**：在你電腦上跑，**不用開 Cursor**，不用花 AI 額度，專門顧 MCP 活著。  
- **Cursor Automations / Copilot Automations（以後可選）**：比較適合掃 repo、分 issue、寫 PR 草稿，**不能取代**本機 MCP 服務。  
  參考：[Copilot Automations 建議清單](https://github.com/Edgar-s-Tool/edgars-cf-workspace/blob/main/.github/COPILOT-AUTOMATIONS.md)

## 常見問題

**Q：我還需要每天叫你檢查嗎？**  
A：不用。routine 的檢查與重啟交給排程；只有「好幾天都好不了」或「要改東西」再叫你。

**Q：會不會一直重啟把我搞煩？**  
A：HealthCheck **只看不動**。AutoRecover 要**連續失敗 3 次**才重啟 MCP，避免偶發閃斷就狂 restart。

**Q：筆電合蓋睡眠怎麼辦？**  
A：醒來、登入後 LoginStart 會補啟動；醒來後幾分鐘內 HealthCheck 也會再跑。

**Q：怎麼關掉自動巡邏？**  
A：

```powershell
cd V:\projects\mcp-handcraft
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-patrol-scheduler.ps1 -Uninstall
```
