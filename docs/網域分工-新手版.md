# 網域分工（新手版）

名字很像，用途不同。先看這張表。

## 一句話記住

| 你要做的事 | 用哪個網址 |
|-----------|-----------|
| Linear 委派給 Hermes Agent | **webhooks.edgars.tools**（有 s） |
| 通用 webhook 收件匣（GitHub / Linear 等，Cloudflare Worker） | **hooks.edgars.tools**（沒有 web 前綴） |
| ChatGPT / Cursor 連 MCP、Linear OAuth | **mcp.edgars.tools** |

## 誰是誰

| 網域 | 背後跑什麼 | 誰會呼叫 | 典型路徑 | 現在能不能用 |
|------|-----------|---------|---------|-------------|
| **mcp.edgars.tools** | mcp-handcraft Python 伺服器（本機 `:8765`）→ Cloudflare Tunnel | ChatGPT、Cursor、瀏覽器做 OAuth | `/mcp`、`/linear/oauth/*`、`/webhook/linear`（只記 log） | ✅ 已上線 |
| **webhooks.edgars.tools** | Tunnel → 本機 linear-orchestrator（`:8645`） | **Linear App** 送 Agent webhook | `/webhooks/linear`、`/healthz` | ✅ 正式 Linear 入口（tunnel + orchestrator 要活著） |
| **hooks.edgars.tools** | Cloudflare Worker **edgar-hooks-inbox**（`cloudflared/hooks-worker`） | 未來各種服務的 webhook | `/health`、`/linear`、`/github` | ✅ DNS + Worker 已上線；**不是** Linear Agent 正式路 |
| **hooks.whoasked.vip** | Cloudflare Worker **edgars-hooks**（`edgars-cf-workspace` Phase 1 骨架） | 測試 / 未來遷移 | `/webhooks/linear` | ⚠️ 骨架：只存 R2/D1，**不會叫 Hermes** |
| **webhook.edgars.tools**（單數） | — | — | — | ❌ **這個網域不存在**（別填） |

## Linear 委派走哪條路

```
Linear 送 webhook
  → https://webhooks.edgars.tools/webhooks/linear
  → Cloudflare Tunnel（edgar-local-01-tunnel）
  → Windows 本機 linear-orchestrator（port 8645）
  → 10 秒內回 Linear → 叫 Hermes 做事
```

**不是**走 `hooks.edgars.tools`，也**不是**走 `mcp.edgars.tools/webhook/linear`（那個只記 log，Linear 會顯示 Did not respond）。

## 常見問題

| 問題 | 答案 |
|------|------|
| Linear 後台 Webhook URL 要填什麼？ | `https://webhooks.edgars.tools/webhooks/linear` |
| hooks.edgars.tools 也要設嗎？ | **不用**，那是另一個通用收件匣 |
| hooks 跟 webhooks 差在哪？ | 差一個字，用途完全不同 |
| 兩個 hooks Worker 是什麼？ | `edgar-hooks-inbox` → `hooks.edgars.tools`；`edgars-hooks` → `hooks.whoasked.vip` |
